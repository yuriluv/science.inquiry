# 시행착오 기록

## v1 (초기, 폐기)
- n_air를 "2.0mL 노출"을 공기 부피로 해석하여 계산 → 오류 발견
- 수압 보정 ρgh를 각 데이터 포인트마다 다른 V_exp로 계산 → V_exp는 실험 고정값이었음
- 냉각 포인트 3,4가 동일 조건(13°C, 2.8mL)인데 중복으로 포함 → 제거 필요

## v2 (δT 단일 scanning, 폐기)
- δT만 scanning, n_air는 초기조건 Dalton 법칙으로 종속 계산
- δT↑ → n_air↑ 피드백 루프 발견: T_gas↓ → P_water_lit↓ → P_air↑ → n_air↑
- 이 피드백으로 냉각 저온 포인트 P_water < 0 모순 심화
- δT ≈ 4.0에서 회귀 불연속 점프 발생 (냉각 포인트 탈락)

## v3 (δT + n_air 독립 scanning)
- δT, n_air_heat, n_air_cool 모두 독립 scanning 파라미터
- P_water > 0 물리적 제약 추가
- V_exp 고정값 수정 (가열=2.0mL, 냉각=2.2mL)
- 냉각 포인트 3,4 중복 제거

### 문제: 냉각 데이터 P_water/P_lit 비율 과대
- Case B: 냉각 20°C에서 P_water/P_lit = 5.49 (문헌값의 5.5배!)
- Case C: 냉각 13°C에서 P_water/P_lit = 0.456 (오히려 과소)
- 냉각 포인트의 P_water 값이 가열 데이터와 전혀 다른 체계에 있음
- R² 하락은 냉각 포인트가 회귀선에서 크게 벗어남을 의미

### 핵심 발견: 트레이드오프
- 가열만(Case A): R²=0.978 높으나 ΔH_vap 오차 10.5%
- 전체(Case C): ΔH_vap 오차 2.6%까지 개선되나 R²=0.213로 붕괴
- 냉각 포인트가 C-C 선형성을 심각하게 훼손

### v4: 앵커 보정 (냉각 0°C 외삽 → 가열 앵커 추가)
